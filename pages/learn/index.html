Photoshop Integration

Framer comes with an application to convert your Photoshop document into a Framer project. This eliminates the need for slicing up your images and rebuilding your views in Framer. To get a quick overview of how this works see this video.

To begin, just open the Framer application, a Photoshop document and click Run. After running you will be presented with a folder containing your project. The project folder contents consists of images, the framework, view data, an index.html and app.js file. Open the index.html file in your browser to see your project.

The app.js file is where you start coding. By default it contains some example code that you can safely remove. Framer added an object called PSD that has references to all your views by name. So if your layer group is called iPhone, you can access it's views with PSD.iPhone or PSD["iPhone"]. The views are normal Framer views, so for example to animate an the view created from the layer group named iPhone you can do: 

  PSD.iPhone.on("click", function() {
    PSD.iPhone.animate({
      properties: {scale: 2.0}
    })
  })

From this point you can use Framer like you normally would to basically add behavior to your Photoshop mock. You can also safely run Framer again after a change in Photoshop, it will update the views but not touch your code.

The rules for converting Photoshop files are:

  • Every layer group will become an ImageView, backed by the pixel and/or shape layers in that layer group.
  • Views are accessible by name under the PSD object.
  • Framer respects the layer hierarchy, so a sub layer group will become a subview. This way you can define your view structure in Photoshop.
  • Layer groups with vector masks will become views with clipping enables. You can optionally add scroll to the layer group name to make it a ScrollView.
  • Layer group names should be unique, if they're not Framer will silently rename them on export.

Debugging

Because Framer runs in the browser you can use all the debugging tools available like the console, timeline, profiler and debugger. Framer will alert you when a Javascript error occurred.

Framer has a few keyboard shortcuts to help with specific debug tasks:

Key Description
Escape (toggle) Switch between the normal view and a debug view of all layers. Useful to inspect structure and positions.
Shift (hold) While shift is pressed, Framer will delay the time by 25x. This will slow down animations and timers like utils.delay and utils.interval. Great to examine animations in depth.

Views

A view is just a square that draws stuff. It has a set of basic properties like x, y, width and height so it knows where to draw. Next to that it has more visual properties like opacity and scale. Views can display images or text, or a combination.

Show Example

View Hierarchy

Views can contain other views. We call these subviews. The parent of a view is called a superview. Views influence their subviews. For example, if you change the opacity of a view all subviews will become opaque too.

A views coordinates are based on it's superview. So with an x and y of zero, a view gets placed in the most top left corner of its superview.

Show Example

Types of Views

Everything is based on a view. But some views have a specific task like scrolling their contents or showing an image. We call these subclasses of a view, for example an ImageView or ScrollView.

It is easy to create components consisting of multiple views that you can re-use across projects, for example a collection view.

Show Example

Animation

Most properties of a view can be animated: x, y, width, height, scale, opacity. You can animate multiple properties at the same time.

Animations optionally take a curve that describes the type of animation. You can used pre defined curves like linear or ease-in, custom ones or spring animations.

You can create an animation explicitly with myAnimation = new Animation({..}) or with the shortcut on any view myView.animate({..}).

Show Example

Styling

Because Framer runs in the browser, you can simply use html and css to style any view. To set the html contents of a view use someView.html = "hello". You can set some text or html.

To add css styling you can edit the properties of the style attribute. You can set one at the time like view.style.backgroundColor = "red" or update multiple properties at the same time as in the example. Please note that while most Framer properties take numeric values, the style attribute requires specific css values like view.style.lineHeight = "16px". You can also set a css class on a view by doing view.class = "myClass". This way you can use an external css file for styling.

Show Example

Events

Events are things that can happen to a view, for example a user click or mouse over, but also a scroll. You can listen to these events and respond to it to build complex interactions.

There are many events you can listen to, but the most interesting are probably click, dblclick, mouseover, mouseout, and keydown. Mobile browsers have specific touch events: touchstart, touchmove and touchend.

To listen to an event use view.on(<event>, <function>). To stop listening you can use view.off(<event>, <function>). You can listen to multiple events at the same time.
