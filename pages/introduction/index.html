Install and Setup

To get started, download Framer and unzip it. From there you use the Framer app to generate a new project from a Photoshop file. Or you can begin from scratch with the template:

  • Open index.html in Safari or Chrome
  • Start editing app.js

Motivation

Many people already prototype in the browser. It's simple and quick. But while html/js/css/jquery gets a lot done it has some downsides:

  • It can get pretty complicated mixing all the different technologies
  • It can be hard to get the pixel perfect control you want
  • It's not always performant, especially on mobile
  • It's pretty far from how it will be actually implemented if you prototype for native

Framer tries to solve some of these problems by providing a very lightweight framework modeled after larger application frameworks. The basic idea is that you only need a few simple building blocks like images, animation and events to build and test complex interactions.

Inspired by Rasmus excellent UILayer, it uses modern webkit technologies to leverage the graphics card, so it runs smooth on both desktop and mobile.

Framer can be seen as an alternative to Flash or Quartz Composer. It's targeted at designers that can do basic javascript, up until experienced interface programmers.

Philosophy

Framer was built to solve an existing practical problem: it's damn hard to add behavior to your mocks. I sincerely hope the future will bring us tools that lets us express interaction more naturally than raw code. But today Framer nicely solves a problem, and that's what it was designed for.

Users

Framer has been tested by a large group of volunteers and is used in the design departments of Facebook, Instagram, Google, Twitter, Dropbox and many other great companies and talented individuals.
