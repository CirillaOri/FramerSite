The core of Framer is a Javascript Framework that enables you to define interaction and animation in a simple, readable way. It's a small .js file that makes creating realistic prototypes a breeze – complete with animations, spring physics and 3D effects. Framer Generator optimizes the flow of going from static mockups to an interactive prototype. It's a desktop application that extracts the resources and layer hierarchy out of Sketch or Photoshop and renders them to a Framer project folder. You can now add interaction to the layers.
### Installation To get started, Download Framer and unzip it. Here's what you get:
  • Project folder
    • app.js
    • index.html
    • Framer folder
    • Images folder 
  • Framer Generator.app
Open index.html and you'll see the logo bounce on a click. Start editing app.js to add interactions and animations. Try it out by switching the function that bounces the logo with this one: 
// Bounce on a click
logoView.on("click", function() {
	logoView.scale = 0.2;
	logoView.animate({
		properties: {scale:1, rotationX:180},
		curve: "spring(600,30,1000)"
	});
});
Congratulations, you have made your first animation with Framer!

### Integration #### Sketch Content
#### Photoshop Framer comes with an application to convert your Photoshop document into a Framer project. This eliminates the need for slicing up your images and rebuilding your views in Framer. To get a quick overview of how this works see this video. To begin, just open the Framer application, a Photoshop document and click Run. After running you will be presented with a folder containing your project. The project folder contents consists of images, the framework, view data, an index.html and app.js file. Open the index.html file in your browser to see your project. The app.js file is where you start coding. By default it contains some example code that you can safely remove. Framer added an object called PSD that has references to all your views by name. So if your layer group is called iPhone, you can access it's views with PSD.iPhone or PSD["iPhone"]. The views are normal Framer views, so for example to animate an the view created from the layer group named iPhone you can do: 
layer = new Layer width:100, height:100
PSD.iPhone.on("click", function() {
  PSD.iPhone.animate({
    properties: {scale: 2.0}
  })
})
From this point you can use Framer like you normally would to basically add behavior to your Photoshop mock. You can also safely run Framer again after a change in Photoshop, it will update the views but not touch your code. The rules for converting Photoshop files are:
  • Every layer group will become an ImageView, backed by the pixel and/or shape layers in that layer group.
  • Views are accessible by name under the PSD object.
  • Framer respects the layer hierarchy, so a sub layer group will become a subview. This way you can define your view structure in Photoshop.
  • Layer groups with vector masks will become views with clipping enabled. You can optionally add "scroll" to the layer group name to make it a ScrollView.
  • Layer group names should be unique, if they're not Framer will silently rename them on export.
